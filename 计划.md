# 培训机构签到与课时管理 APP — 开发计划

**版本**：v1.0
**日期**：2026-03-10
**平台**：Android + SQLite + ECharts

---

## 总览

| 步骤 | 名称 | 核心内容 | 依赖 |
|---|---|---|---|
| Step 1 | 基础框架 + 数据层 | 数据库、登录、学生管理 | 无 |
| Step 2 | 签到核心功能 | 点名、拍照、状态标记、课时扣除 | Step 1 |
| Step 3 | 课时管理 + 跟进列表 | 续费记录、三色预警、待跟进列表 | Step 1 |

## Step 1：基础框架 + 数据层

### 开发内容

- **数据库初始化**：创建 SQLite 数据库，建立以下数据表：
  - `students`（学生基本信息 + 状态 + 剩余课时）
  - `purchases`（购课/续费记录）
  - `attendances`（签到记录）
  - `settings`（系统配置，如预警阈值）
  - `users`（管理员 + 老师 PIN 码）
- **PIN 码登录**：管理员/老师各自独立 PIN，登录后进入对应权限页面
- **学生管理（管理员）**：
  - 新增学生（姓名、班级、家长、手机、首次购课日期/数量/金额）
  - 编辑学生基本信息
  - 设置学生状态（在读 / 退课）
  - 学生列表展示（仅在读学生）、归档学生可查询
- **基础导航结构**：底部 Tab 或侧边菜单，区分老师视图和管理员视图

### 交付物

- 可运行的 Android APK
- SQLite 数据库完整建表脚本
- 登录页面（PIN 输入 + 角色跳转）
- 学生列表页、新增/编辑学生页

### 验收标准

- [ ] 管理员 PIN 登录后可进入管理界面，老师 PIN 登录后进入老师界面，PIN 错误无法登录
- [ ] 管理员可成功新增一名学生，所有字段均保存正确
- [ ] 学生列表只显示"在读"状态的学生，退课学生从列表消失但可在归档中查询
- [ ] 编辑学生信息后，列表数据实时更新
- [ ] 老师账号进入后，无法看到新增/编辑学生的入口

## Step 2：签到核心功能

### 开发内容

- **今日签到页面**：展示所有在读学生，默认状态为「到课」
- **签到状态标记**：支持对每位学生单独设置：到课 / 迟到 / 请假（可加备注）/ 旷课
- **班级合影拍照**：签到时拍摄一张合影，压缩后与当日签到记录绑定存储
- **课时扣除逻辑**：
  - 到课、迟到 → 自动扣 1 课时
  - 请假、旷课 → 不扣课时
- **今日签到统计**：页面顶部实时显示 应到 / 实到 / 请假 / 旷课 / 迟到 人数及当日出勤率
- **签到记录修改**（仅管理员）：可修改历史记录，修改操作留存审计日志（修改时间 + 改前状态 + 改后状态）

### 交付物

- 今日签到页面（含状态标记 + 实时统计）
- 拍照功能（压缩存储 + 与签到记录关联）
- 课时自动扣除逻辑
- 管理员签到记录修改页（含审计日志）

### 验收标准

- [ ] 签到页面正确显示所有在读学生
- [ ] 标记「到课」或「迟到」后，该学生剩余课时减少 1，标记「请假」「旷课」不变
- [ ] 页面顶部统计数字随标记操作实时更新
- [ ] 请假备注可正常保存并在记录中查看
- [ ] 合影照片拍摄后正确压缩存储，可在该日签到详情中查看
- [ ] 管理员修改历史签到状态后，课时相应变更，审计日志正确记录
- [ ] 老师无法看到签到记录的修改入口

## Step 3：课时管理 + 跟进列表

### 开发内容

- **课时三色预警**（学生列表和签到列表均显示）：
  - 🟢 绿色：剩余课时 > 阈值
  - 🟡 黄色：剩余课时 ≤ 阈值
  - 🔴 红色：剩余课时 = 0
- **预警阈值设置**（管理员）：在系统设置中配置提醒天数（如 3 天）
- **续费录入**（管理员）：新增续费记录（日期 / 数量 / 金额 / 备注），自动累加到剩余课时
- **购课/续费历史**（管理员）：按时间线查看某学生的完整购买记录
- **待跟进列表**（老师 + 管理员）：
  - 自动汇总黄色 + 红色预警学生
  - 每条记录显示：姓名 / 剩余课时 / 预警状态 / 跟进状态
  - 跟进状态：未联系 → 已联系 → 已续费（续费后自动移出列表）
  - 可添加跟进备注

### 交付物

- 学生列表课时颜色标注（绿/黄/红）
- 预警阈值设置页
- 续费录入页 + 购买历史页
- 待跟进列表页（含跟进状态流转）

### 验收标准

- [ ] 剩余课时超过阈值显示绿色，达到阈值显示黄色，为 0 显示红色
- [ ] 修改预警阈值后，学生颜色状态立即重新计算
- [ ] 管理员录入续费后，剩余课时正确累加，购买历史中出现新记录
- [ ] 黄色和红色学生自动出现在待跟进列表，绿色学生不出现
- [ ] 跟进状态可从「未联系」流转至「已联系」再至「已续费」，「已续费」后移出列表
- [ ] 老师无法看到购买历史和续费录入入口

## Step 4：统计分析（ECharts）

### 开发内容

- **出勤日历视图**（老师 + 管理员）：
  - 周视图：本周每天出勤人数，点击查看当天详情（到课/请假/旷课/迟到各有谁）
  - 月视图：本月每天出勤人数，点击查看当天详情
- **出勤统计报表**（老师 + 管理员，ECharts 图表）：
  - 月出勤趋势折线图（按日展示出勤率）
  - 按人统计：出勤率排名、迟到次数、请假次数、旷课次数
- **经营统计报表**（仅管理员，ECharts 图表）：
  - 月度收入柱状图（每月新购 + 续费金额）
  - 课时消耗速度：每位学生平均每周消耗课时数 + 预计耗尽日期
  - 续费转化：课时用完的学生中，续费 vs 流失比例（饼图）
  - 学生留存：本月在读人数 vs 上月对比
  - 生命周期分布饼图（新生 / 活跃 / 预警 / 流失风险各有多少人）

> ECharts 通过 WebView 内嵌方式在 Android 中渲染

### 交付物

- 周视图 / 月视图日历页（含日期详情弹窗）
- 出勤统计报表页（ECharts 折线图 + 列表）
- 经营统计报表页（ECharts 柱状图 + 饼图，仅管理员可见）

### 验收标准

- [ ] 周视图和月视图正确显示每天出勤人数，点击日期弹出当天详情
- [ ] 月出勤趋势折线图数据与签到记录一致
- [ ] 按人出勤统计数据正确（出勤率 = 实到次数 / 应到次数）
- [ ] 经营报表的月度收入与续费录入记录金额吻合
- [ ] 课时耗尽日期预测逻辑正确（剩余课时 ÷ 周均消耗速度）
- [ ] 老师登录后看不到经营统计入口
- [ ] ECharts 图表在 Android WebView 中正常渲染，无白屏/报错

## Step 5：数据导出 + 备份 + 整体联调

### 开发内容

- **Excel 导出**（管理员）：
  - 按学生导出：签到历史 + 课时记录
  - 按月导出：当月所有学生出勤汇总
  - 导出文件保存至手机本地存储
- **数据备份与还原**（管理员）：
  - 一键导出 SQLite 数据库文件至本地 / SD 卡
  - 支持从备份文件还原数据
  - 系统显示「距上次备份已 X 天」提示
- **生命周期标签完善**：确认新生 / 活跃 / 预警 / 流失风险标签在各页面正确联动
- **全流程联调测试**：
  - 从新增学生 → 签到 → 课时扣除 → 预警 → 续费 → 跟进 → 统计 完整链路验证
  - 边界情况测试（课时为 0 时签到、修改历史记录后统计是否同步等）
- **性能与体验优化**：签到页面加载 < 1 秒，图表渲染流畅

### 交付物

- Excel 导出功能（按学生 / 按月）
- 数据备份 / 还原功能
- 完整 APK 安装包（最终交付版本）
- 简要操作说明文档

### 验收标准

- [ ] 导出的 Excel 文件可在手机本地找到，用 Excel 打开数据完整无乱码
- [ ] 备份文件导出成功后，删除 APP 数据，使用备份文件还原，数据完整恢复
- [ ] 系统正确显示距上次备份天数
- [ ] 完整流程测试：新增学生 → 首次购课 → 连续签到至课时耗尽 → 出现红色预警 → 进入跟进列表 → 续费后恢复绿色 → 统计图表数据正确
- [ ] 课时为 0 的学生在签到列表中红色置顶显示
- [ ] 修改历史签到后，统计报表数据同步更新
- [ ] 签到页面打开到可操作时间 < 1 秒（50 名学生数据量下测试）
