# Android 测试矩阵（UI / 代码 / 业务）

## 1. 测试目标
- 代码质量：构建可用、静态检查通过、单元测试通过。
- UI 可用性：核心页面可访问、关键交互无阻塞、异常态可见。
- 业务正确性：签到、拍照、历史、导出、备份恢复链路正确。

## 2. 前置条件（必须）
- 已安装 JDK 17+，并设置 `JAVA_HOME`。
- 已安装 Android SDK Platform-Tools，`adb` 可用。
- 真机已开启开发者模式与 USB 调试。
- 真机通过 `adb devices` 可识别。

## 3. 自动化层（代码）
- `npm run build`
  - 通过标准：无 TypeScript / Vite 构建错误。
- `npm run cap:sync:android`
  - 通过标准：Capacitor 同步成功，无插件错误。
- `npm run android:assemble:debug`
  - 通过标准：生成 debug APK。
- `npm run android:lint`
  - 通过标准：无阻塞级 lint 错误（可留低优先警告）。
- `npm run android:test:unit`
  - 通过标准：Gradle unit tests 全绿。

## 4. 真机层（UI）
- 登录页
  - 输入错误 PIN 提示正确；正确 PIN 可进入首页。
- 今日签到页
  - 学生列表完整显示，提交按钮不遮挡末项。
  - 到课/迟到/请假按钮状态切换正确。
  - 拍照按钮可拉起相机，拍照后状态正确更新。
- 历史页
  - 日期切换、记录展示、合影查看流程正常。
- 设置页
  - PIN 修改流程、导出/备份入口可用。

## 5. 真机层（业务）
- 业务链路 A：签到提交
  - 前置：当日未提交。
  - 操作：拍照 -> 标记状态 -> 提交。
  - 预期：当日记录写入；已提交提示出现；历史页可见。
- 业务链路 B：课时扣减
  - 预期：到课/迟到扣 1，请假不扣；统计口径正确。
- 业务链路 C：导出与恢复
  - 导出文件可生成；恢复后数据一致（抽查 3 名学生 + 2 天签到）。
- 业务链路 D：边界与异常
  - 无学生、无照片、重复提交、权限拒绝时提示正确且不会崩溃。

## 6. 结果记录模板
- 设备：品牌/型号/Android 版本
- 构建号：提交号 + 打包时间
- 用例编号：A/B/C/D + 步骤
- 结果：Pass/Fail
- 证据：截图/日志/导出文件
- 缺陷：严重级别 + 复现步骤 + 影响范围
