# 培训机构学生签到与课时管理系统 — 需求文档

**版本**：v1.0
**日期**：2026-03-10
**平台**：Android 单机 APP（SQLite 本地数据库）

---

## 一、项目概述

为培训机构提供一套学生签到与课时管理工具，核心目标：

1. 记录学生每次到课情况，留存签到凭证（防止后期纠纷）
2. 管理每位学生的课时数量，实时掌握剩余课时
3. 提供出勤与经营两个维度的统计分析，辅助管理决策
4. 支持续费跟进，降低学生流失率

## 三、功能模块详述

### 3.1 学生管理

#### 3.1.1 学生信息录入（管理员）

每位学生包含以下基本信息：

| 字段 | 说明 |
|---|---|
| 姓名 | 必填 |
| 班级 | 必填 |
| 家长姓名 | 必填 |
| 家长手机号 | 必填 |
| 首次购课日期 | 必填 |
| 首次课时数量 | 必填（即剩余课时初始值） |
| 首次缴费金额 | 必填（手动输入，系统不做价格规则） |
| 学生状态 | 在读 / 退课（默认在读） |

#### 3.1.2 学生状态管理

- **在读**：出现在每日点名列表和所有统计中
- **退课**：从活跃列表移除，进入归档，历史数据完整保留，可查询

#### 3.1.3 学生生命周期标签（系统自动计算）

系统根据课时状态和续费行为，自动为每位学生打标：

| 标签 | 触发条件 |
|---|---|
| 新生 | 首次购课 30 天内 |
| 活跃 | 正常在读，课时充足 |
| 预警 | 课时即将用完（达到提醒阈值） |
| 流失风险 | 课时已清零，超过 X 天未续费 |

---

### 3.2 签到管理

#### 3.2.1 签到状态定义

每次上课，每位学生有且仅有一个签到状态：

| 状态 | 是否扣课时 | 备注 |
|---|:---:|---|
| 到课 | ✅ 扣 1 课时 | 正常出勤 |
| 迟到 | ✅ 扣 1 课时 | 标记为迟到，计入出勤统计 |
| 请假 | ❌ 不扣 | 可添加文字备注，标记区分 |
| 旷课 | ❌ 不扣 | 标记留档，计入旷课统计 |

#### 3.2.2 每日签到操作流程

1. 老师打开 APP，进入「今日签到」页面
2. 系统显示所有**在读**学生列表
3. 老师拍摄一张**班级大合影**，与当日签到记录绑定存档
4. 老师逐一标记每位学生的签到状态（支持批量默认"到课"，再单独标记异常）
5. 对请假学生可输入备注（如"发烧请假"）
6. 提交签到，系统自动扣除相应课时

#### 3.2.3 签到记录修改

- 仅管理员可修改历史签到记录
- 修改后系统自动记录"修改时间 + 修改前状态 + 修改后状态"，留存审计痕迹

#### 3.2.4 签到统计视图（老师 + 管理员可见）

- **今日视图**：应到人数 / 实到人数 / 请假人数 / 旷课人数 / 迟到人数 / 当日出勤率
- **周视图**：以日历形式展示本周每天出勤情况，点击具体日期查看当天详情
- **月视图**：以日历形式展示本月每天出勤情况，点击具体日期查看当天详情

---

### 3.3 课时管理

#### 3.3.1 课时规则

- 课时无使用期限，用完为止，不设冻结
- 每次「到课」或「迟到」签到，自动扣除 1 课时
- 「请假」和「旷课」不扣课时

#### 3.3.2 课时三色预警

系统根据剩余课时数自动显示颜色，老师和管理员均可见数字和颜色：

| 颜色 | 触发条件 | 含义 |
|---|---|---|
| 🟢 绿色 | 剩余课时 > 提醒阈值 | 课时充足 |
| 🟡 黄色 | 剩余课时 ≤ 提醒阈值 | 即将用完，需跟进续费 |
| 🔴 红色 | 剩余课时 = 0 | 课时已耗尽，不得上课 |

- 提醒阈值由管理员在系统设置中配置（如提前 3 天提醒）
- 红色学生在签到列表中置顶显示

#### 3.3.3 续费记录（管理员操作）

每次续费作为一条独立记录保存，字段如下：

| 字段 | 说明 |
|---|---|
| 续费日期 | 必填 |
| 续费课时数 | 必填（累加到剩余课时） |
| 缴费金额 | 必填（手动输入） |
| 备注 | 选填 |

- 支持查看某学生的完整购买/续费历史（时间线形式）
- 系统维护「剩余课时 = 历次购买总量 - 已消耗课时数」，确保账目准确

---

### 3.4 待跟进列表

#### 3.4.1 列表触发条件

以下学生自动出现在「待跟进列表」中：

- 剩余课时达到黄色预警阈值（即将用完）
- 剩余课时为 0（已用完未续费）

#### 3.4.2 跟进状态（老师和管理员均可操作）

| 状态 | 说明 |
|---|---|
| 未联系 | 默认状态 |
| 已联系 | 已电话/沟通，等待回复 |
| 已续费 | 完成续费，自动移出列表 |

- 已续费后系统自动将该学生移出待跟进列表
- 可记录跟进备注（如"家长说下周来续"）

---

### 3.5 统计分析

#### 3.5.1 出勤统计（老师 + 管理员）

| 维度 | 内容 |
|---|---|
| 按日 | 每天出勤率、各状态人数 |
| 按月 | 本月出勤趋势折线图、月均出勤率 |
| 按人 | 每位学生的出勤率排名、迟到次数、请假次数、旷课次数 |

#### 3.5.2 经营统计（仅管理员）

| 维度 | 内容 |
|---|---|
| 月度收入 | 每月新购 + 续费金额汇总 |
| 课时消耗速度 | 每位学生平均每周消耗课时数 |
| 预计耗尽日期 | 根据消耗速度预测该学生课时用完的大概日期 |
| 续费转化 | 课时用完的学生中，续费 vs 流失的比例 |
| 学生留存 | 本月在读人数 vs 上月对比 |
| 生命周期分布 | 新生 / 活跃 / 预警 / 流失风险 各有多少人 |

---

### 3.6 数据管理（仅管理员）

#### 3.6.1 数据导出

- 支持导出 Excel 文件
- 可按学生、按月、按出勤维度选择导出内容
- 导出文件保存至手机本地存储

#### 3.6.2 数据备份

- 支持一键备份，将数据库文件导出至手机本地 / SD 卡
- 建议管理员定期手动备份（系统可提示"距上次备份已 X 天"）

## 四、技术要求

| 项目 | 要求 |
|---|---|
| 平台 | Android 原生 APP |
| 数据库 | SQLite（本地单机存储） |
| 图表库 | ECharts（统计分析页面所有图表均使用 ECharts 渲染） |
| 图片存储 | 合影照片压缩后存储至本地，与签到记录关联 |
| 登录方式 | PIN 码（管理员 PIN / 老师 PIN 各自独立） |
| 网络要求 | 无需联网，完全离线运行 |

---

## 五、非功能性需求

- **性能**：学生列表、签到操作响应时间 < 1 秒
- **易用性**：签到流程不超过 3 步，老师无需培训即可上手
- **数据安全**：PIN 码本地加密存储，防止未授权访问
- **可靠性**：签到提交后立即写库，防止 APP 意外退出丢失数据
